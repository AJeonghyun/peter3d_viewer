# 갈릴리 2D 월드 콘셉트

기존 3D 월드를 바로 교체하기 전에 방향을 확인하기 위한 독립 HTML 시안입니다.
메인 React 애플리케이션과 데이터는 변경하지 않습니다.

## 시안에서 확인할 수 있는 것

- 갈릴리 호수와 베드로의 어부 이야기를 담은 3단 횡스크롤 맵
- 25개 조를 위에서부터 9명, 8명, 8명으로 분산 배치
- 모든 캐릭터의 자동 좌우 이동과 방향 전환
- 캐릭터 터치 시 선택 강조, 이동 정지, 조 정보 패널 표시
- 레벨, 달란트, 용기, 현명, 진실, 열정 표시
- `우리 조 찾기` 버튼으로 4조 캐릭터 바로 선택
- iPad 1024×768과 작은 화면 대응
- 동작 줄이기와 투명 효과 줄이기 접근성 설정 대응

## 실행

프로젝트 루트에서 다음 명령을 실행합니다.

```bash
python3 -m http.server 8768 --directory prototypes/galilee-2d-concept
```

브라우저에서 `http://127.0.0.1:8768/`을 엽니다.

## 실제 서비스로 전환할 때

현재 시안은 가장 빠르게 화면 방향을 확인하기 위해 HTML, CSS 애니메이션,
JavaScript만 사용합니다. 25명이 단순히 세 줄을 걷고 터치되는 현재 범위라면
React와 CSS 스프라이트만으로 충분하며, Three.js와 물리 엔진을 제거할 수 있어
iPad의 발열과 메모리 사용량도 크게 줄일 수 있습니다.

점프, 사다리 이동, 발판 충돌, 카메라 스크롤, 미니게임까지 추가한다면
React 안에서 2D 게임 캔버스를 분리하고 Phaser 또는 PixiJS 기반으로 옮기는
방식이 적합합니다. 이 결정은 단순 월드 전환을 먼저 검증한 뒤 내려도 됩니다.

## 조별 캐릭터 제작 방식

1. 학생 그림을 업로드합니다.
2. 배경을 제거해 투명 PNG 캐릭터를 만듭니다.
3. 측면 걷기 포즈 4장과 대기 포즈 2장을 생성합니다.
4. 같은 크기의 스프라이트 시트 또는 개별 WebP로 저장합니다.
5. 조 데이터의 `model_url` 대신 2D 스프라이트 URL을 연결합니다.
6. 월드에서는 조별 이미지와 능력치 데이터만 불러옵니다.

초기 행사 버전은 대기 1장과 걷기 2장만으로도 충분합니다. 25조 전체를 3D GLB로
읽는 것보다 파일 크기, 로딩 시간, 제작 비용을 훨씬 낮출 수 있습니다.
